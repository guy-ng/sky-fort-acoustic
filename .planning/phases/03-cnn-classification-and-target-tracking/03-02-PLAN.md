    **2. Create `src/acoustic/tracking/schema.py`** (per D-08, D-09):
    - `class EventType(str, Enum)`: `NEW = "new"`, `UPDATE = "update"`, `LOST = "lost"`
    - `class TargetEvent(BaseModel)`:
      ```python
      event: EventType
      target_id: str        # UUID string
      class_label: str      # "drone" or "background"
      confidence: float     # 0.0-1.0, drone probability
      az_deg: float         # azimuth bearing
      el_deg: float         # elevation bearing
      speed_mps: float | None = None  # null until ms-2 (D-07)
      timestamp: float      # time.monotonic() or time.time()
      ```

    **3. Create `src/acoustic/tracking/events.py`** (per D-10 — WebSocket broadcast, NOT ZeroMQ):
    - `class EventBroadcaster`:
      - Uses `asyncio.Queue` subscribers pattern (same pattern as DeviceMonitor in existing codebase).
      - `__init__(self)` — empty set of subscriber queues.
      - `subscribe(self) -> asyncio.Queue` — creates and returns a new queue, adds to subscribers set.
      - `unsubscribe(self, queue: asyncio.Queue) -> None` — removes queue from subscribers set.
      - `broadcast(self, event: TargetEvent) -> None` — synchronous method that puts `event.model_dump()` into all subscriber queues using `queue.put_nowait()`. Catches `asyncio.QueueFull` and drops (non-blocking). This method is called from the pipeline thread, so it must be thread-safe. Use a lock around the subscribers set iteration.
      - No ZeroMQ, no pyzmq. Pure asyncio.Queue fanout for WebSocket consumers.

    **4. Create `src/acoustic/tracking/tracker.py`**:
    - `@dataclass class TrackedTarget`:
      ```python
      id: str               # UUID4 string
      class_label: str      # "drone"
      az_deg: float
      el_deg: float
      confidence: float
      speed_mps: float | None = None  # always None for now (D-07)
      created_at: float     # time.monotonic()
      last_seen: float      # time.monotonic()
      lost: bool = False
      ```
    - `class TargetTracker`:
      - `__init__(self, ttl: float = 5.0, broadcaster: EventBroadcaster | None = None)` — ttl is seconds before target marked lost (D-06). Optional broadcaster for event publishing.
      - `update(self, az_deg: float, el_deg: float, confidence: float, class_label: str = "drone") -> TrackedTarget`:
        - If no active (non-lost) target exists: create new TrackedTarget with `uuid.uuid4()`, store it, emit `TargetEvent(event=EventType.NEW, ...)` via broadcaster.
        - If active target exists: update bearing, confidence, last_seen. Emit `TargetEvent(event=EventType.UPDATE, ...)` via broadcaster.
        - Returns the TrackedTarget.
        - NOTE: Single-target tracking for Phase 3 (multi-target is future enhancement per research open questions). Only one active target at a time.
      - `tick(self) -> list[str]`:
        - Check all active targets. If `time.monotonic() - target.last_seen > ttl`, mark as lost, emit `TargetEvent(event=EventType.LOST, ...)` via broadcaster. Return list of lost target IDs.
      - `get_active_targets(self) -> list[TrackedTarget]`:
        - Return list of non-lost targets.
      - `get_target_states(self) -> list[dict]`:
        - Return list of dicts compatible with TargetState model (id, class_label, speed_mps, az_deg, el_deg, confidence). Used by /ws/targets and /api/targets endpoints.
