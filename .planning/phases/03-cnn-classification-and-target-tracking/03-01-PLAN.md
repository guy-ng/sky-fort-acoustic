    Functions (all must be importable by name):
    - `fast_resample(y: np.ndarray, fs_in: int, fs_out: int) -> np.ndarray` — uses `scipy.signal.resample_poly` with `math.gcd` for up/down factors. Returns float32.
    - `make_melspec(y: np.ndarray, sr: int) -> np.ndarray` — calls `librosa.feature.melspectrogram(y=y, sr=sr, n_fft=N_FFT, hop_length=HOP_LENGTH, n_mels=N_MELS, power=2.0)`, then `librosa.power_to_db(S, ref=np.max)`, returns `S_db.T` (shape: frames x n_mels).
    - `pad_or_trim(spec: np.ndarray, max_frames: int = MAX_FRAMES) -> np.ndarray` — pads with zeros or truncates first axis to `max_frames`.
    - `norm_spec(spec: np.ndarray) -> np.ndarray` — `(spec - mean) / (std + 1e-8)`.
    - `preprocess_for_cnn(mono_audio: np.ndarray, fs_in: int) -> np.ndarray` — full pipeline: resample to 16kHz, take/pad last 2s, make_melspec, pad_or_trim, norm_spec, reshape to `(1, 128, 64, 1)` float32.

    **4. Create `src/acoustic/classification/inference.py`**:
    - `class OnnxDroneClassifier`:
      - `__init__(self, model_path: str)` — loads ONNX model via `ort.InferenceSession(model_path, providers=["CPUExecutionProvider"])`. Stores input name from `session.get_inputs()[0].name`. Raises `FileNotFoundError` if path doesn't exist (check before loading). Logs model loaded at INFO level.
      - `predict(self, preprocessed: np.ndarray) -> float` — runs `session.run(None, {input_name: preprocessed})`, returns `float(outputs[0].flat[0])` clamped to [0.0, 1.0].

    **5. Create `src/acoustic/classification/state_machine.py`**:
    - `class DetectionState(str, Enum)`: `NO_DRONE = "NO_DRONE"`, `CANDIDATE = "DRONE_CANDIDATE"`, `CONFIRMED = "DRONE_CONFIRMED"`
    - `class DetectionStateMachine`:
      - `__init__(self, enter_threshold: float = 0.80, exit_threshold: float = 0.40, confirm_hits: int = 2)` — per D-05, thresholds are Claude's discretion. Use POC values as defaults.
      - `state: DetectionState` property — current state, starts as NO_DRONE.
      - `update(self, drone_probability: float) -> DetectionState` — implements 3-state hysteresis:
        - NO_DRONE: if prob >= enter_threshold -> CANDIDATE, hit_count=1
        - CANDIDATE: if prob >= enter_threshold -> hit_count+=1, if hit_count >= confirm_hits -> CONFIRMED. If prob <= exit_threshold -> NO_DRONE, reset hit_count.
        - CONFIRMED: if prob <= exit_threshold -> CANDIDATE, reset hit_count. (Note: goes to CANDIDATE not directly NO_DRONE, giving one more chance)
      - `reset(self) -> None` — resets to NO_DRONE, hit_count=0.
