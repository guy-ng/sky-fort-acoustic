## Decisions Made

- **Eval row formula correction (D-17 amendment):** The locked CONTEXT.md D-17 says `parquet_row_count == len(rglob source_dir, "*.wav"))`. Reading `scripts/export_uma16_parquet.py:export_eval` confirms eval rows are written from `labels.json` entries, not from `rglob`. The plan implements D-17's *intent* (row-drift gate) with the correct eval-side formula `count_eval_label_entries(labels.json)`. Same user-facing behavior (test fails on drift), correct implementation. Flag for `/gsd-discuss-phase` only if the user wants the literal D-17 wording amended.
- **Inlined literal regen command** rather than f-string interpolation through `REGEN_HINT`, so a plain text grep on the file source finds every fail-path's hint (defensive against future refactors that might rename the constant and break the implicit grep contract).
- **No exporter import** (D-18): constants are duplicated verbatim. The duplication is the price of guaranteeing the preflight can never grow an auto-regen capability by accidentally importing into the exporter module.
