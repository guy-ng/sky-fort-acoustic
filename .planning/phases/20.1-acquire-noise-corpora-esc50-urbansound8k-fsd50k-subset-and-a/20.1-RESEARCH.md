## Summary

- **soundata 1.0.1 is verified and appropriate for all three corpora.** The top-level API is `soundata.initialize(name, data_home=None, version="default")` followed by `.download(partial_download=None, force_overwrite=False, cleanup=False)` and `.validate(verbose=True)`. Default cache is `~/sound_datasets/<name>/`. `data_home` is a plain string/path override and is the cleanest way to land downloads in a project-local directory. ESC-50 and UrbanSound8K are single-remote downloads (one tarball each). FSD50K has 5 remote keys: `FSD50K.dev_audio`, `FSD50K.eval_audio`, `ground_truth`, `metadata`, `documentation` — `partial_download` accepts a subset but Phase 20.1 needs dev_audio + eval_audio + ground_truth at minimum.
- **All 6 D-18 FSD50K classes verified present in vocabulary.csv** with exact AudioSet mids. The labels are stored in `FSD50K.ground_truth/dev.csv` and `eval.csv` with schema `fname,labels,mids,split` (dev) / `fname,labels,mids` (eval). The `labels` column is comma-separated; "primary tag" = `labels.split(",")[0]`.
- **FSD50K is distributed at 44.1 kHz mono**, not 16 kHz. `BackgroundNoiseMixer.warm_cache()` already resamples on load (verified in `augmentation.py`), so no resampling step is needed in the acquisition script — but the class-filter COPY step preserves 44.1 kHz WAVs on disk. Document this so the planner doesn't plan a spurious resample task.
- **The single biggest planning risk** is FSD50K's `data_home` behavior combined with the temp-to-final copy requirement (D-08). Cleanest pattern: set `data_home=<temp_dir>` (e.g., `data/noise/.fsd50k_tmp/`) in the acquisition script, run `.download()`, parse `dev.csv`+`eval.csv` to build the per-class file list, copy to `data/noise/fsd50k_subset/{class}/`, then `shutil.rmtree(temp_dir)` in a `finally`. Do NOT use soundata's default `~/sound_datasets` — it's off-repo and would leave 24 GB of orphaned cache on the host after cleanup.
- **The parquet sync check is simpler than it looks.** `data/parquet/{ambient,eval}/train-0.parquet` are single files produced by `scripts/export_uma16_parquet.py`. Row count comes from `pyarrow.parquet.read_metadata(path).num_rows` (fast — no table load). mtime comes from `os.stat(path).st_mtime` (the file itself, not parquet metadata — the columnar file's own mtime is the freshest truth). Sources: `AMBIENT_SRC = REPO_ROOT / "data/field/uma16_ambient"`, `EVAL_SRC = REPO_ROOT / "data/eval/uma16_real"`. Eval rows count is determined by `labels.json` entries (one per WAV), not by `rglob` — see §Parquet sync semantics for the exact formula.

**Primary recommendation:** Structure the acquisition script as one `argparse` dispatcher with four subcommands (`esc50`, `urbansound8k`, `fsd50k`, `all`). For each corpus: (1) check `.acquired.json` marker → skip if matches, (2) `disk_usage` guard, (3) `soundata.initialize(name, data_home=<staging dir inside data/noise>)`, (4) `download()` inside `try/finally`, (5) for FSD50K only: class-filter copy step, (6) write `.acquired.json`, (7) for FSD50K only: `rmtree` staging dir in `finally`. Write the preflight as ONE file (`test_noise_corpora_present.py`) with a `test_parquet_shards_in_sync` function at the bottom — these tests share the same `ACOUSTIC_SKIP_NOISE_PREFLIGHT` module-level skip gate, and the planner avoids duplicating the env var handling. Use `pytest.fail(msg)` not bare `assert` for actionable D-15 messages.

## soundata API reference

### Top-level API — `soundata.__init__`

```python
# From soundata/__init__.py (verified against main branch, 2026-04-07)
def initialize(dataset_name, data_home=None, version="default"):
    """Load a soundata dataset by name."""
```

- `dataset_name`: string, e.g. `"esc50"`, `"urbansound8k"`, `"fsd50k"`.
- `data_home`: `str | None`. **If `None`, defaults to `~/sound_datasets/<name>/` (or `/tmp/sound_datasets/<name>/` if `HOME` is unset).** This is the critical planning fact — the script MUST pass `data_home` explicitly to keep downloads inside the repo.
- `version`: defaults to `"default"`, which maps to the current stable index version in each loader. Leave as default for all three corpora.

Returns: a `core.Dataset` subclass instance exposing `download()`, `validate()`, `clip_ids`, plus per-corpus properties.

### `Dataset.download()` — core.py:310

```python
def download(self, partial_download=None, force_overwrite=False, cleanup=False):
    """Download data to `save_dir` and optionally print a message.

    Args:
        partial_download (list or None):
            A list of keys of remotes to partially download.
            If None, all data is downloaded
        force_overwrite (bool):
            If True, existing files are overwritten by the downloaded files.
        cleanup (bool):
            Whether to delete any zip/tar files after extracting.

    Raises:
        ValueError: if invalid keys are passed to partial_download
        IOError: if a downloaded file's checksum is different from expected
    """
```

- **Returns:** `None` on success (the files land in `data_home`; you check presence via the filesystem, not a return value).
- **Exceptions:** `ValueError` (bad partial_download keys), `IOError` (checksum mismatch), plus whatever `download_utils.downloader` re-raises from `requests`/`urllib` — typically `URLError`, `HTTPError`, `ConnectionError` on network failure. The script should wrap `.download()` in a single `try/except Exception` that logs the failing corpus and re-raises with a clear "check your network / Zenodo availability" message.
- **`cleanup=True` is recommended for FSD50K** — it deletes the multi-part zip files (`FSD50K.dev_audio.zip` + `.z01..z05`) after extraction, saving ~8 GB of intermediate disk during the filter step.
- **`force_overwrite=False` is the idempotency lever on the soundata side.** Combined with the `.acquired.json` marker (D-03) the script has two layers of skip: marker present → skip entirely; marker absent → call `download()`, which internally skips already-extracted files.

### `Dataset.validate()` — core.py:378

```python
def validate(self, verbose=True):
    """Validate if the stored dataset is a valid version

    Returns:
        * list - files in the index but are missing locally
        * list - files which have an invalid checksum
    """
```

Returns a tuple `(missing_files, invalid_checksums)`. Both empty → dataset is valid. **The acquisition script should call `validate()` after `download()` and abort if either list is non-empty**, with a message like `"soundata.validate() for esc50 reports N missing, M invalid checksums — re-run with --force to re-download"`.

### FSD50K — soundata/datasets/fsd50k.py

Five remote keys:

| Key | What | Size | Multi-part |
|-----|------|------|------------|
| `FSD50K.dev_audio` | Dev split audio (40,966 clips) | ~23 GB | YES — 6 parts (`.zip` + `.z01..z05`) |
| `FSD50K.eval_audio` | Eval split audio (10,231 clips) | ~3.7 GB | YES — 2 parts (`.zip` + `.z01`) |

Multi-part remotes: soundata's `download_utils` handles the `.zip + .z01..z0N` reconstruction automatically via a `zip` subprocess (hence the note in the source: "remotes that need to be extracted together should be grouped in a list"). **This requires the host to have `zip` installed** — on macOS/Linux it's usually present; document as a dependency in the script header.

**Partial download recommendation:** `partial_download=["FSD50K.dev_audio", "FSD50K.eval_audio", "ground_truth"]`. Skip `metadata` (the per-clip Freesound titles/descriptions aren't needed for class filtering — only `ground_truth/dev.csv`+`eval.csv`+`vocabulary.csv` matter) and `documentation` (LICENSE text, small but unused by the script).

**On-disk layout after extraction (inside `data_home`):**
```
<data_home>/
├── FSD50K.dev_audio/          # 40,966 WAVs, named by Freesound ID (e.g. "64760.wav")
├── FSD50K.eval_audio/         # 10,231 WAVs
└── FSD50K.ground_truth/
    ├── dev.csv                # columns: fname,labels,mids,split
    ├── eval.csv               # columns: fname,labels,mids
    └── vocabulary.csv         # columns: index,label,mid (200 rows)
```

**Audio format:** "PCM 16 bit 44.1 kHz mono" per the upstream docstring — **not** 16 kHz. `BackgroundNoiseMixer.warm_cache()` in `src/acoustic/training/augmentation.py` resamples on load, so the on-disk 44.1 kHz WAVs are correct and no resampling step is needed in the acquisition script.

**Exceptions under `download()` for FSD50K:** same as base `download()` plus a much higher probability of network interruption (24 GB over 8 files). The script should NOT retry automatically — a crash leaves the partial download in place, and the next invocation resumes via soundata's per-file skip behavior (checksums are verified on existing files before re-download).

### CSV parsing — `dev.csv` and `eval.csv`

Schemas (verified in `soundata/datasets/fsd50k.py:load_ground_truth`):

```
dev.csv:  fname,labels,mids,split
eval.csv: fname,labels,mids
```

- `fname` — Freesound ID, no extension. The on-disk file is `FSD50K.dev_audio/<fname>.wav` or `FSD50K.eval_audio/<fname>.wav`.
- `labels` — comma-separated slugs, e.g. `"Wind,Rain"` or `"Bird"`. Smeared (propagated up the ontology).
- `mids` — comma-separated AudioSet mids in the same order as `labels`.
- `split` (dev only) — `train` or `val`. Ignored for Phase 20.1 — all dev clips are fair game as training noise.

**Primary tag rule (D-09):** `primary = row["labels"].split(",")[0]`. First element of the comma-separated list. No ranking is implied by the order in the CSV — it's just stable so the first entry is a deterministic choice.

```python
import csv

def iter_fsd50k_clips(ground_truth_csv: Path, audio_dir: Path):
    """Yield (wav_path, primary_label) for every clip whose primary label is in FSD50K_TARGET_CLASSES."""
    with ground_truth_csv.open() as fh:
        reader = csv.DictReader(fh)
        for row in reader:
            primary = row["labels"].split(",", 1)[0].strip()
            if primary in FSD50K_TARGET_CLASSES:
                wav_path = audio_dir / f"{row['fname']}.wav"
                if wav_path.is_file():
                    yield wav_path, primary
```

## Disk + idempotency patterns

### Disk-space guard (D-10)

```python
import shutil
from pathlib import Path

REQUIRED_FREE_GB_FSD50K = 30  # D-10 threshold; dev+eval temp (24 GB) + subset copy headroom

def check_disk_space(path: Path, required_gb: int, corpus: str) -> None:
    path.mkdir(parents=True, exist_ok=True)
    free_bytes = shutil.disk_usage(path).free
    free_gb = free_bytes / (1024 ** 3)
    if free_gb < required_gb:
        raise RuntimeError(
            f"Insufficient disk space for {corpus}: "
            f"{free_gb:.1f} GB free at {path}, need >= {required_gb} GB. "
            f"Free some space and re-run."
        )
```

**30 GB is correct for D-10.** FSD50K peak disk = ~24 GB download (8 zip parts simultaneously) + ~27 GB extracted audio on top briefly before `cleanup=True` removes zips → ~27 GB peak extracted. Add the per-class subset copy (~2–4 GB) and the total peak is ~29 GB. 30 GB is the minimum; a planner who wants to be safer could bump to 35 GB — Claude's discretion.

**Cheaper corpora:** ESC-50 needs ~2 GB (650 MB zip + 1.3 GB extracted). US8K needs ~14 GB (6 GB tarball + 7 GB extracted). Guard each corpus with its own threshold, not a single 30 GB number applied to all three.

### `.acquired.json` marker schema (D-03)

Recommended fields:

```json
{
  "schema_version": 1,
  "corpus": "esc50",
  "source": "soundata:esc50@1.0",
  "soundata_version": "1.0.1",
  "file_count": 2000,
  "total_bytes": 1398340000,
  "acquired_at": "2026-04-07T14:23:11Z"
}
```

- `schema_version: int` — include it. If the script ever grows a new field (e.g., `sha256` of the WAV tree), the preflight test can gracefully handle older markers.
- `corpus: str` — matches the subcommand name.
- `source: str` — encodes where it came from, e.g. `"soundata:esc50@1.0"` or `"soundata:fsd50k@1.0+filter:6class-primary"` for FSD50K.
- `soundata_version: str` — captured via `soundata.__version__` or `importlib.metadata.version("soundata")`. Useful if a future soundata release changes a loader's on-disk layout.
- `file_count: int` — count of `.wav` files under the corpus dir. The preflight reads this for diagnostic output.
- `total_bytes: int` — sum of WAV file sizes. Cheap to compute, catches partial re-downloads.
- `acquired_at: str` — ISO 8601 UTC.

**Idempotency check:**
```python
def is_already_acquired(corpus_dir: Path) -> bool:
    marker = corpus_dir / ".acquired.json"
    if not marker.is_file():
        return False
    data = json.loads(marker.read_text())
    actual_count = sum(1 for _ in corpus_dir.rglob("*.wav"))
    return actual_count == data.get("file_count")
```

Skip if marker exists AND the live WAV count matches the marker. If they diverge (e.g., someone deleted files manually), re-run — which is what the user presumably wants.

## Pytest preflight patterns

### Existing repo convention (verified)

`tests/integration/test_vertex_dockerfile_copy.py` is the canonical pattern reference. It uses:
- No fixtures (zero dependency on `conftest.py` — the file has a `running_app` async fixture for lifespan tests, but vertex_dockerfile_copy ignores it).
- Module-level `DOCKERFILE = Path(__file__).resolve().parents[2] / "Dockerfile.vertex"`.
- A private `_read_dockerfile()` helper that calls `pytest.skip(...)` if the file isn't found.
- `test_*` functions are plain `def`, take no args, use `assert ... , "<actionable message>"`.

**Deviation for Phase 20.1:** `test_vertex_dockerfile_copy.py` uses `pytest.skip` when the target file is absent. **D-12 explicitly forbids this pattern** — the new preflight must HARD FAIL on missing data. Use `pytest.fail(msg)` instead of `pytest.skip(msg)`. The ONLY skip path is the env-var opt-out (see below).

### Env-var opt-out idiom (D-12)

Use **module-level** skip gate, not per-test:

```python
# tests/integration/test_noise_corpora_present.py
"""Preflight: verify noise corpora exist on disk (Phase 20.1 D-11..D-15).

This test is a hard gate. It MUST fail on missing data so Phase 20-05
can never silently ship a Docker image without background-noise training data.

The single auditable opt-out is the env var ACOUSTIC_SKIP_NOISE_PREFLIGHT=1.
CI may set it explicitly; the training host MUST NOT.
"""

from __future__ import annotations

import os
from pathlib import Path

import pytest
import soundfile as sf

if os.environ.get("ACOUSTIC_SKIP_NOISE_PREFLIGHT") == "1":
    pytest.skip(
        "ACOUSTIC_SKIP_NOISE_PREFLIGHT=1 — noise corpora preflight skipped",
        allow_module_level=True,
    )

REPO_ROOT = Path(__file__).resolve().parents[2]
NOISE_ROOT = REPO_ROOT / "data" / "noise"
```

**Why module-level:** One grep for `ACOUSTIC_SKIP_NOISE_PREFLIGHT` finds all skip sites — the whole file skips or none of it does. Per-test skip scatters the opt-out across multiple `pytest.skip()` calls and loses the auditability promise of D-12.

`allow_module_level=True` is REQUIRED when calling `pytest.skip()` at import time — without it, pytest raises "Using pytest.skip outside of a test is not allowed" and the whole collection fails.

### `pytest.fail` vs `assert` for actionable messages (D-15)

**Recommendation: `pytest.fail(msg)`.** Rationale:

| Pattern | Output on failure |
|---------|-------------------|
| `assert count >= 2000, "data/noise/esc50/ has 0 wav files. Run: ..."` | Traceback + line number + AssertionError + message. Noisy. |
| `if count < 2000: pytest.fail("data/noise/esc50/ has 0 wav files. Run: ...")` | Clean "FAILED" line with just the message. |

`pytest.fail()` produces a `Failed` exception (not `AssertionError`) and pytest renders it as a one-line failure with no traceback. For D-15 the goal is "the failure message points at the fix" — cleaner output wins.

### Per-corpus assertion pattern (D-13 + D-14)

```python
def _count_wavs(corpus_dir: Path) -> int:
    return sum(1 for _ in corpus_dir.rglob("*.wav"))


def _spot_check_readable(corpus_dir: Path, corpus: str, n: int = 2) -> None:
    """D-14: soundfile.info on 1-2 files to catch truncated downloads."""
    import random
    wavs = list(corpus_dir.rglob("*.wav"))
    if not wavs:
        return  # count check will catch this
    sample = random.sample(wavs, min(n, len(wavs)))
    for p in sample:
        try:
            info = sf.info(str(p))
        except Exception as e:
            pytest.fail(
                f"{corpus}: {p.relative_to(REPO_ROOT)} failed to read: {e}. "
                f"Download may be truncated. Re-run: "
                f"python scripts/acquire_noise_corpora.py {corpus} --force"
            )
        if info.frames == 0:
            pytest.fail(
                f"{corpus}: {p.relative_to(REPO_ROOT)} is zero-frame. "
                f"Re-run: python scripts/acquire_noise_corpora.py {corpus} --force"
            )


def test_esc50_present() -> None:
    corpus_dir = NOISE_ROOT / "esc50"
    count = _count_wavs(corpus_dir)
    if count < 2000:
        pytest.fail(
            f"data/noise/esc50/ has {count} wav files (expected >= 2000). "
            f"Run: python scripts/acquire_noise_corpora.py esc50"
        )
    _spot_check_readable(corpus_dir, "esc50")


def test_urbansound8k_present() -> None:
    corpus_dir = NOISE_ROOT / "urbansound8k"
    count = _count_wavs(corpus_dir)
    if count < 8000:
        pytest.fail(
            f"data/noise/urbansound8k/ has {count} wav files (expected >= 8000). "
            f"Run: python scripts/acquire_noise_corpora.py urbansound8k"
        )
    _spot_check_readable(corpus_dir, "urbansound8k")


FSD50K_CLASSES = (
    "Wind", "Rain", "Traffic_noise_and_roadway_noise",
    "Mechanical_fan", "Engine", "Bird",
)


def test_fsd50k_subset_present() -> None:
    corpus_dir = NOISE_ROOT / "fsd50k_subset"
    total = _count_wavs(corpus_dir)
    if total < 1500:
        pytest.fail(
            f"data/noise/fsd50k_subset/ has {total} wav files (expected >= 1500). "
            f"Run: python scripts/acquire_noise_corpora.py fsd50k"
        )
    for cls in FSD50K_CLASSES:
        cls_dir = corpus_dir / cls
        cls_count = _count_wavs(cls_dir)
        if cls_count < 100:
            pytest.fail(
                f"data/noise/fsd50k_subset/{cls}/ has {cls_count} wav files (expected >= 100). "
                f"Run: python scripts/acquire_noise_corpora.py fsd50k --force"
            )
    _spot_check_readable(corpus_dir, "fsd50k_subset")
```

**Runtime budget:** D-14 says <1 second total. `rglob` over ~11k files is O(10ms) on SSD; `sf.info` is metadata-only (reads header, not PCM) so it's ~1ms per call. Total well under 100ms for all three corpora + spot checks.

## Parquet sync semantics

### Source-dir constants (from `scripts/export_uma16_parquet.py`)

Verified by reading the file (2026-04-07):

```python
REPO_ROOT = Path(__file__).resolve().parent.parent
AMBIENT_SRC = REPO_ROOT / "data" / "field" / "uma16_ambient"
EVAL_SRC = REPO_ROOT / "data" / "eval" / "uma16_real"
EVAL_LABELS_PATH = EVAL_SRC / "labels.json"
OUT_ROOT = REPO_ROOT / "data" / "parquet"

# Shard paths:
# data/parquet/ambient/train-0.parquet   (from AMBIENT_SRC via rglob("*.wav"))
# data/parquet/eval/train-0.parquet      (from EVAL_SRC via labels.json entries)
```

**Row semantics (verified):**
- **Ambient shard:** one row per `*.wav` under `AMBIENT_SRC` (recursive rglob, sorted). All labels = 0. Row count = `len(list(AMBIENT_SRC.rglob("*.wav")))`.
- **Eval shard:** one row per entry in `EVAL_SRC/labels.json`. Each entry has `{"file": "<relative path>", "label": "drone"|"no_drone"}`. The script verifies each referenced WAV exists. **Row count = `len(json.load(EVAL_LABELS_PATH))`, NOT `rglob("*.wav")` on `EVAL_SRC`** — because `labels.json` may reference a subset of the WAVs present on disk.

**Planning implication for D-17:** The CONTEXT.md D-17 says `parquet_row_count == len(rglob source_dir, "*.wav")`. This is correct for the **ambient** shard but WRONG for the **eval** shard. The planner must split the sync check into two branches:

```python
# Ambient branch
ambient_wavs = sum(1 for _ in AMBIENT_SRC.rglob("*.wav"))
ambient_rows = pq.read_metadata(AMBIENT_PARQUET).num_rows
assert ambient_wavs == ambient_rows

# Eval branch
eval_entries = len(json.loads(EVAL_LABELS_PATH.read_text()))
eval_rows = pq.read_metadata(EVAL_PARQUET).num_rows
assert eval_entries == eval_rows
```

This is a **research-surfaced correction to D-17**. Flag in the plan — the locked decision language needs a minor amendment to reference `labels.json` for the eval side. This is not re-opening the decision; it's implementing D-17's spirit (sync check) against the actual export semantics of `export_uma16_parquet.py`.

### Row count via `pyarrow.parquet.read_metadata` (fast)

```python
import pyarrow.parquet as pq

def parquet_row_count(path: Path) -> int:
    return pq.read_metadata(str(path)).num_rows
```

`read_metadata()` reads only the footer (~few KB) — no columns are decoded. Works on both shards in <10ms.

### mtime gate — file-level, not parquet-metadata

```python
import os

def parquet_mtime(path: Path) -> float:
    return os.stat(path).st_mtime

def max_source_mtime(src_dir: Path) -> float:
    return max((os.stat(p).st_mtime for p in src_dir.rglob("*.wav")), default=0.0)
```

**Recommendation: use filesystem mtime on the parquet file itself.** Parquet metadata *does* include a `created_by` string in the footer but no reliable timestamp — the file's own `st_mtime` is the freshest truth and is what the build pipeline cares about ("was the shard written after the last WAV modification?").

**Failure-message pattern for D-17:**

```python
if parquet_rows != ambient_wavs:
    pytest.fail(
        f"data/parquet/ambient/train-0.parquet has {parquet_rows} rows but "
        f"data/field/uma16_ambient/ has {ambient_wavs} wav files. "
        f"Shard is stale. Regenerate: "
        f"python scripts/export_uma16_parquet.py --ambient-only"
    )

if parquet_mtime < max_src_mtime:
    pytest.fail(
        f"data/parquet/ambient/train-0.parquet is older than the newest source WAV in "
        f"data/field/uma16_ambient/. Regenerate: "
        f"python scripts/export_uma16_parquet.py --ambient-only"
    )
```

### Shard glob — actually single files

CONTEXT.md phrases the shards as `train-*.parquet` (plural glob). The current export writes a single `train-0.parquet` per dataset (verified in `export_uma16_parquet.py:71` and `:110`). The glob form is forward-compatible — Phase 20.1 should still iterate `list((OUT_ROOT/"ambient").glob("train-*.parquet"))` (sum row counts across all matching shards) so if a future plan shards the export across multiple files the sync check doesn't break.

```python
def all_shard_paths(shard_dir: Path) -> list[Path]:
    return sorted(shard_dir.glob("train-*.parquet"))

def total_rows(shard_dir: Path) -> int:
    return sum(pq.read_metadata(str(p)).num_rows for p in all_shard_paths(shard_dir))

def newest_shard_mtime(shard_dir: Path) -> float:
    return max((os.stat(p).st_mtime for p in all_shard_paths(shard_dir)), default=0.0)
```

If `all_shard_paths(shard_dir)` is empty, fail with "shard file missing — regenerate via scripts/export_uma16_parquet.py" — this is the direct analog to D-12's "empty data = hard failure" for the parquet side.

| Pattern | STRIDE | Standard Mitigation |

## Open questions for the planner

1. **Per-corpus staging location vs single staging dir.** The recommendation above uses `data/noise/.fsd50k_stage/` as a hidden staging dir for FSD50K. Should ESC-50 and US8K ALSO download into hidden staging dirs first, then move into their final homes on success? Pro: atomicity — a partial download doesn't leave a half-populated `data/noise/esc50/`. Con: 2x disk during the move (for US8K's 7 GB this is non-trivial). **Recommendation:** keep ESC-50 and US8K direct (soundata's existing file-level resume handles partials); only FSD50K needs staging because of the class-filter transform. **Planner decision:** accept or override.

2. **`Mechanical_fan` class count fallback.** §FSD50K class filtering flags `Mechanical_fan` as AT RISK for the `>=100` threshold under primary-label-only filtering. If the real count comes in below 100, the script needs a fallback: either (a) relax to secondary-label matching for that one class (`Mechanical_fan` appears anywhere in `labels`), or (b) drop `Mechanical_fan` from D-13's per-class assertion and document the gap. **Recommendation:** implement (a) as a warning-logged fallback — the test still asserts `>=100` but the script's copy step searches secondary labels if primary yields too few. **Planner decision:** confirm the fallback is acceptable, or route back to the user as a CONTEXT amendment.

3. **Parquet eval-shard row count formula.** §Parquet sync semantics documents that D-17's `parquet_row_count == len(rglob source_dir, "*.wav")` is correct for the ambient shard but not the eval shard — eval uses `labels.json` as the source of truth. This is a research-surfaced correction, not a scope change. **Planner decision:** lock the corrected formula in the plan and note it as a CONTEXT.md amendment candidate (or route to `/gsd-discuss-phase` for explicit user approval — but this is a straightforward bug fix to the decision text, not a design change).

4. **`cleanup=True` vs preserving FSD50K zips.** Soundata's `download(cleanup=True)` deletes the 8 zip parts after extraction. This saves ~24 GB but means re-running the script re-downloads from Zenodo. Alternative: `cleanup=False` plus a separate `--cleanup` CLI flag the operator runs once they're satisfied. **Recommendation:** `cleanup=True` is correct for Phase 20.1 — the `.acquired.json` marker is the idempotency layer, not the zip files, and 24 GB of orphaned zips on the training host is a real disk-space cost. **Planner decision:** accept or add a `--keep-archives` flag for Claude's discretion.

5. **Conftest fixtures for the preflight test.** The existing `tests/integration/conftest.py` has a `running_app` fixture that imports `acoustic.main` and starts the app lifespan. The new preflight test does NOT need this fixture (no app, just filesystem checks) but by virtue of living in `tests/integration/` it could accidentally trigger conftest imports. **Recommendation:** verify `tests/integration/conftest.py` does not have module-level side effects that fail when the simulated device isn't configured — read it once during planning and confirm. Line 11 sets `os.environ["ACOUSTIC_AUDIO_SOURCE"] = "simulated"` as a safe module-level side effect; the fixture itself is opt-in via parameter. **Planner decision:** no conftest change needed; document for confidence.

6. **`make data-noise` shim — include or skip?** Claude's discretion per CONTEXT.md. **Recommendation:** skip. The single-script pattern (D-01) is deliberately UX-forward, and a `make` shim adds a second entry point that can drift. If the operator wants a shorter command, `alias data-noise='python scripts/acquire_noise_corpora.py all'` in their shell rc works equally well. **Planner decision:** accept skip or include as a small nicety.
