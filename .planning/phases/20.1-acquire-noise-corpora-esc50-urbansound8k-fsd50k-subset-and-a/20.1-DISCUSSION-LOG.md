## Download mechanism

### Q1: How should the noise corpora be acquired on a fresh machine?

| Option | Description | Selected |
|--------|-------------|----------|
| One Python script | scripts/acquire_noise_corpora.py with subcommands per corpus, idempotent, uses soundata. Single entrypoint, matches existing scripts/ pattern. | ✓ |
| Makefile target | `make data-noise` wrapping wget/unzip/python. Adds new Makefile to repo. | |
| Per-corpus shell scripts | scripts/data/fetch_*.sh — 3 entrypoints to maintain. | |
| README-only manual steps | Doc-only. Defeats "never silently happen again" goal. | |

**User's choice:** One Python script (recommended)

### Q2: How should UrbanSound8K's registration be handled?

| Option | Description | Selected |
|--------|-------------|----------|
| Detect-and-instruct | Script detects user-placed tarball, otherwise prints form URL + path. | |
| Fully scripted with token | Use a direct download path with no manual form step. | ✓ (resolved into Q3) |
| README pointer only | No script handling, manual placement. | |

**User's choice:** Fully scripted — clarified in Q3 below.

### Q3: How should US8K actually be fetched in the script?

| Option | Description | Selected |
|--------|-------------|----------|
| soundata urbansound8k loader | soundata.initialize('urbansound8k').download() — Zenodo direct URL, no token needed, same dep as FSD50K. | ✓ |
| Direct Zenodo curl in script | Hardcoded URL + sha256, no soundata dep, more brittle. | |
| HF Hub mirror | Unofficial mirror provenance. | |

**User's choice:** soundata urbansound8k loader (recommended)
**Notes:** soundata's loader bypasses the courtesy registration form by hitting the official Zenodo direct URL.

## Preflight test design

### Q1: What should tests/integration/test_noise_corpora_present.py assert?

| Option | Description | Selected |
|--------|-------------|----------|
| Min file count + readability | Per-corpus wav count thresholds + 1-2 random files opened with soundfile. <1s runtime. | ✓ |
| Min file count only | Just count thresholds. Misses corrupt downloads. | |
| Full duration/SR audit | Walk every file. Slow, belongs in a separate slow-marker test. | |

**User's choice:** Min file count + readability (recommended)

### Q2: Should the preflight verify all 6 FSD50K classes?

| Option | Description | Selected |
|--------|-------------|----------|
| Yes — per-class min count | Each of 6 D-18 classes has >=N files. Catches silently-dropped class. | ✓ |
| No — just total count | Single total for fsd50k_subset/. Misses missing class. | |

**User's choice:** Yes — per-class min count (recommended)

### Q3: Where should the preflight test live?

| Option | Description | Selected |
|--------|-------------|----------|
| Integration test, default suite | tests/integration/test_noise_corpora_present.py in normal pytest run. | ✓ |
| Marked slow / opt-in | @pytest.mark, only with --runslow. Forgotten before docker build. | |
| Standalone preflight script | Not pytest, separate script. Looser integration. | |

**User's choice:** Integration test, default suite (recommended)

### Q4: On a machine where data/noise/ is empty, should the test skip or fail?

| Option | Description | Selected |
|--------|-------------|----------|
| Always fail | No skip path. Strongest guarantee, matches roadmap intent. CI without data is RED unless it sets the explicit opt-out env var. | ✓ |
| Skip on CI, fail locally | Skip if CI=true. Lighter CI burden, risks silent activation on training host. | |
| Skip only via explicit env var | Test fails by default; skip only via ACOUSTIC_SKIP_NOISE_PREFLIGHT=1. Auditable via grep. | |

**User's choice:** Always fail (recommended)
**Notes:** Captured in CONTEXT.md D-12 with the auditable opt-out env var ACOUSTIC_SKIP_NOISE_PREFLIGHT=1 as the single sanctioned skip mechanism.

## FSD50K acquisition path

### Q1: How should the FSD50K subset be acquired?

| Option | Description | Selected |
|--------|-------------|----------|
| soundata + post-filter | Full FSD50K download (~24 GB peak), then class-filter copy of 6 D-18 classes, then delete temp. | ✓ |
| soundata partial_download | Smaller disk peak (~3-4 GB) using soundata's class subset arg. Slightly brittle to API changes. | |
| Manual Zenodo zip + custom filter | Hardcoded URLs + custom CSV parser. No soundata dep. More code, more brittle. | |

**User's choice:** soundata + post-filter (recommended)
**Notes:** Captured in CONTEXT.md D-07..D-10 with disk-free guard (~30 GB) and try/finally cleanup.

### Q2: How strict should FSD50K class mapping be (multi-label vs single)?

| Option | Description | Selected |
|--------|-------------|----------|
| Single dominant class per file | Use FSD50K's primary label; cleaner per-class dirs, simpler counts. | ✓ |
| Multi-label (copy into all matching classes) | Hardlinks/symlinks across class dirs. Inflates counts. | |
| Flat dir + manifest CSV | Single fsd50k_subset/ dir + CSV manifest. Mixer rglob's anyway. | |

**User's choice:** Single dominant class per file (recommended)

## Parquet shard re-sync

### Q1: How should we verify data/parquet/{ambient,eval} are still in sync with the WAV trees?

| Option | Description | Selected |
|--------|-------------|----------|
| Row count + mtime check | parquet rowcount == wav count AND parquet mtime >= newest wav mtime. Cheap, deterministic. | ✓ |
| Row count only | Just count match. Misses edited-but-same-count case. | |
| Auto-regenerate on drift | Preflight calls export_uma16_parquet.py. Convenient but hides drift. | |
| Content hash manifest | sha256 of every WAV vs stored manifest. Strongest, slowest. | |

**User's choice:** Row count + mtime check (recommended)
**Notes:** Drift surfaces as a hard test failure pointing at scripts/export_uma16_parquet.py for manual re-run (D-18). No auto-regeneration — the human must see drift events.
