# Phase 20.1: Acquire noise corpora ESC50 / UrbanSound8K / FSD50K subset and add host preflight test - Context

**Gathered:** 2026-04-07
**Status:** Ready for planning

<domain>
## Phase Boundary

Acquire three noise corpora (ESC-50, UrbanSound8K, FSD50K subset) onto disk under `data/noise/{esc50,urbansound8k,fsd50k_subset}/` and add a host-side preflight test that fails loudly if any corpus is missing or empty — so Phase 20-05 Task 2 (Vertex submit) can be unblocked and the same silent failure can never recur.

This phase delivers DATA + a TEST GATE. It does NOT modify the trainer, BackgroundNoiseMixer, Dockerfile.vertex-base, or any Phase 20 plan code. It does NOT retrain or submit anything to Vertex — that is the resumed Phase 20 work.

</domain>

<decisions>
## Implementation Decisions

### Acquisition script
- **D-01:** Single Python script `scripts/acquire_noise_corpora.py` with subcommands per corpus (`esc50`, `urbansound8k`, `fsd50k`, plus an `all` convenience target). Idempotent — re-running with data already present is a no-op. Matches the existing `scripts/` pattern (`vertex_submit.py`, `export_uma16_parquet.py`). No Makefile, no shell script sprawl.
- **D-02:** Add `soundata>=1.0,<2.0` to `requirements.txt` (host-side dev dep, not baked into the Docker image — the image only needs the resulting WAV trees, not the downloader).
- **D-03:** Script writes a small `data/noise/<corpus>/.acquired.json` marker per corpus on success containing `{corpus, version/source, file_count, total_bytes, acquired_at}`. Used both for idempotency (skip re-download if marker present and file count matches) and for the preflight test's diagnostic output.

### ESC-50 acquisition
- **D-04:** Use `soundata.initialize('esc50').download()` then move/symlink the resulting tree into `data/noise/esc50/` so `BackgroundNoiseMixer.rglob("*.wav")` finds all 2000 clips. License: CC BY-NC 4.0 (acceptable for research/training; document in script header).

### UrbanSound8K acquisition
- **D-05:** Use `soundata.initialize('urbansound8k').download()` — soundata's loader hits the Zenodo direct URL, no registration form / token needed. The Urban Sound dataset registration form is courtesy, not a hard gate. ~6 GB tarball, then extract into `data/noise/urbansound8k/`.
- **D-06:** Do NOT use HF Hub mirrors or unofficial mirrors — provenance must be Zenodo (official source).

### FSD50K acquisition
- **D-07:** Use `soundata.initialize('fsd50k').download()` to fetch the full FSD50K dev+eval set into a temp location (~24 GB peak disk).
- **D-08:** After download, run a Python class-filter step that copies (not symlinks — the resulting tree must survive `COPY data/noise` into Docker) the 6 D-18 classes into `data/noise/fsd50k_subset/{class}/<file>.wav`, then deletes the temp.
- **D-09:** Class-to-file assignment: **single dominant class per file**. Use FSD50K's `ground_truth/dev.csv` and `ground_truth/eval.csv` `labels` column primary tag. A clip labeled both `Wind` and `Rain` lands in `Wind/` if `Wind` is the primary tag. Cleaner per-class dirs, simpler preflight count. Exact matching of the 6 class slugs → AudioSet ontology IDs is in the Planner's discretion (use the official `FSD50K.ground_truth/vocabulary.csv`).
- **D-10:** Disk peak ~25 GB during the FSD50K step. Script must check available disk before starting and abort with a clear error if <30 GB free. Temp dir cleanup is in a `try/finally` so a crash mid-filter doesn't leave 24 GB orphaned.

### Preflight test
- **D-11:** Test file: `tests/integration/test_noise_corpora_present.py`. Lives in the default `pytest` suite (no opt-in marker). Runs as part of `pytest tests/integration/`.
- **D-12:** **Always fails on missing data — no skip path.** Empty `data/noise/<corpus>/` (only `.gitkeep` present) is a hard test failure regardless of host (CI, fresh clone, training host). Strongest guarantee; matches roadmap intent ("can never silently happen again"). The single, auditable opt-out is `ACOUSTIC_SKIP_NOISE_PREFLIGHT=1` env var — CI sets it explicitly if it needs to skip; the training host never sets it; `grep ACOUSTIC_SKIP_NOISE_PREFLIGHT` finds every skip site.
- **D-13:** Per-corpus assertions:
  - **ESC-50:** `len(rglob("*.wav")) >= 2000`
  - **UrbanSound8K:** `len(rglob("*.wav")) >= 8000` (real count is 8732)
  - **FSD50K subset:** `len(rglob("*.wav")) >= 1500` AND each of the 6 D-18 classes (`Wind`, `Rain`, `Traffic_noise_and_roadway_noise`, `Mechanical_fan`, `Engine`, `Bird`) has `>=100` files in its per-class dir.
- **D-14:** Readability spot-check: for each corpus, pick 1–2 random `.wav` files and call `soundfile.info(path)` to confirm they are not zero-byte / corrupt. Catches truncated downloads. Total runtime <1 second.
- **D-15:** Failure messages must point at the fix: `"data/noise/esc50/ has 0 wav files. Run: python scripts/acquire_noise_corpora.py esc50"` — not just an `assert` traceback.

### Parquet shard re-sync verification
- **D-16:** Add a sibling preflight test `tests/integration/test_parquet_shards_in_sync.py` (or extend the noise-corpora test file — Planner's discretion) that verifies `data/parquet/{ambient,eval}/train-*.parquet` are still consistent with their WAV source trees in `data/field/uma16_ambient/` and `data/eval/uma16_real/`.
- **D-17:** Sync check semantics: **row count match + mtime gate**.
  - `parquet_row_count == len(rglob source_dir, "*.wav"))`
  - `max(parquet.mtime) >= max(source_wav.mtime)`
  - Both must hold. Either failing → test fails with a message pointing at `scripts/export_uma16_parquet.py` as the regeneration command.
- **D-18:** Do NOT auto-regenerate on drift. Drift must surface as a visible test failure so the human knows shards went stale (and so a stale shard never silently ships in `Dockerfile.vertex-base`).
- **D-19:** This phase does NOT touch the WAV trees in `data/field/uma16_ambient/` or `data/eval/uma16_real/`. The check is a sanity gate; if drift is detected the human re-runs `scripts/export_uma16_parquet.py` themselves before re-attempting Phase 20-05 Task 2.

### Claude's Discretion
- Exact CLI surface of `scripts/acquire_noise_corpora.py` (argparse subcommands vs. click; flags for `--data-dir`, `--force`, `--dry-run`).
- Exact FSD50K class-slug → AudioSet ontology ID mapping (must use `FSD50K.ground_truth/vocabulary.csv` as ground truth; capture the resulting mapping in a constant at the top of the script).
- Whether to split the preflight into one test file or two (`test_noise_corpora_present.py` + `test_parquet_shards_in_sync.py`) — Planner can choose based on test fixture sharing.
- Disk free threshold for the FSD50K guard (D-10 says 30 GB but Planner can bump if there's a known reason).
- Conftest fixtures, pytest markers, CI yaml integration.
- Whether to also bake a `make data-noise` shim or `pyproject.toml` script entrypoint as a UX nicety.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **`soundata`** library (NEW dep): provides `esc50`, `urbansound8k`, `fsd50k` loaders with `download()` + `validate()` methods. Verified in Phase 20 RESEARCH §A2 (soundata 1.0.1).
- **`BackgroundNoiseMixer.__init__`** (`src/acoustic/training/augmentation.py:124`) already does `d.rglob("*.wav")` so the on-disk layout is forgiving — per-class subdirs under `fsd50k_subset/` are fine; flat is also fine. No mixer change needed.
- **`scripts/export_uma16_parquet.py`** is the canonical "what should be in the parquet shards" definition — the sync test reuses its source-dir constants.
- **`tests/integration/test_vertex_dockerfile_copy.py`** is an example of how Phase 20-00 wrote integration tests under `tests/integration/` — same pattern (no fixtures, just `Path(...).read_text()` style assertions).

### Established Patterns
- **`scripts/`** directory contains all developer/operator entry points (`vertex_submit.py`, `export_uma16_parquet.py`, etc.) — `acquire_noise_corpora.py` belongs there.
- **`data/`** is gitignored except for `.gitkeep` placeholders and a few small example files (e.g. `data/eval/uma16_real/labels.json.example`). The acquired WAV trees stay out of git — only the `.gitkeep` placeholders and any new `.acquired.json` markers are committed.
- **Integration tests under `tests/integration/`** run as part of the default `pytest` suite (no opt-in markers in current files).
- **`requirements.txt`** is the host-side dependency list; `requirements-vertex.txt` is the trainer-image list. `soundata` belongs in `requirements.txt` only.

### Integration Points
- **`Dockerfile.vertex-base`** consumes `data/noise/` via `COPY` — once this phase populates the dirs, the next `docker build -f Dockerfile.vertex-base` produces a working base image and Phase 20-05 Task 2 can proceed.
- **`scripts/vertex_submit.py`** env var `ACOUSTIC_TRAINING_NOISE_DIRS` already lists the 4 paths — no change needed there once the host dirs are populated.
- **The preflight test** is the new gate — it should run BEFORE `docker build` (whether via `pytest tests/integration/` in a developer workflow or as a pre-build hook step in `Dockerfile.vertex-base`'s build wrapper script).

</code_context>

<specifics>
## Specific Ideas

- "This can never silently happen again" — every decision should be evaluated against this. The preflight test is the structural mechanism, not contributor discipline. That's why D-12 has no skip path and D-15 mandates actionable failure messages.
- Single-script UX (D-01) is the inverse of the Phase 20-00 pattern of "manual capture step" — the lesson from the 20-05 blocker is that "manual" steps get forgotten, even by the same person who wrote them.

</specifics>
