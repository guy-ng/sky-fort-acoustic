# Phase 20.1 Plan 02: Noise Corpora Preflight Summary

**Hard preflight gate that fails `pytest tests/integration/test_noise_corpora_present.py` whenever ESC-50, UrbanSound8K, or the FSD50K 6-class subset is missing or under-populated. Every failure message tells the operator the exact `python scripts/acquire_noise_corpora.py <corpus>` command to run.**

## Accomplishments

- 4-test integration preflight: `test_esc50_present`, `test_urbansound8k_present`, `test_fsd50k_subset_present`, `test_readability_spot_check`
- 7-test meta-test suite that exercises every message helper in isolation via `importlib.util.spec_from_file_location`, with grep guards for the env-var name and `allow_module_level=True` literal
- All 6 D-13 FSD50K class dirs gated (Wind, Rain, Traffic_noise_and_roadway_noise, Mechanical_fan, Engine, Bird) — per-class count < 100 triggers a named failure with `--force` regen command
- D-14 readability spot-check picks 1–2 random WAVs per corpus and runs `sf.info()` (WAV header only, ~44 bytes per call); zero-frame or exception triggers `pytest.fail` with the `--force` re-acquisition command
- Module-level `ACOUSTIC_SKIP_NOISE_PREFLIGHT=1` opt-out keyed to the same env var Plan 20.1-03 uses — one grep finds every preflight skip site in the repo
- **Memory-safe by construction:** no full WAV reads, no PCM decode, no accumulation of binary data into Python lists. The plan was selected over the parquet-regen path specifically to avoid the memory profile that crashed the dev host earlier in this phase.

## Files Created/Modified

## Decisions Made

- **Meta-test import strategy**: Meta-tests load the preflight file via `importlib.util.spec_from_file_location`, popping `ACOUSTIC_SKIP_NOISE_PREFLIGHT` before `exec_module` so the module-level skip does not abort the load. This lets the unit tests exercise the helpers regardless of the env var state and regardless of whether the corpora exist.
- **`sf.info()` over full-file `sf.read()`**: D-14 readability check reads only WAV header metadata, not PCM samples. This is a deliberate memory choice — catching truncated downloads without loading any audio data into RAM. Called out here because a prior dev-host memory crash in this phase was traced to a different script that loaded full WAV bodies into a Python list.
- **No app imports**: The preflight file deliberately does not import anything under `acoustic.*` or `src.*`. The existing `tests/integration/conftest.py` imports `acoustic.main` at module load, but that is a pre-existing project pattern and does not affect the preflight test semantics (the tests themselves only touch the filesystem).

## Recommended Operator Workflow

On the training host, before resuming Phase 20 Plan 20-05 Task 2:

```bash
# Ensure the opt-out is unset
unset ACOUSTIC_SKIP_NOISE_PREFLIGHT

# Run both preflights together (Plan 20.1-02 + Plan 20.1-03)
pytest tests/integration/test_noise_corpora_present.py \
       tests/integration/test_parquet_shards_in_sync.py

# Only if both green, proceed to docker build
docker build -f Dockerfile.vertex-base -t acoustic-vertex-base:v7 .
```

If `test_noise_corpora_present.py` fails, the failure message names the exact `python scripts/acquire_noise_corpora.py <corpus>` command. Run that, then re-run the preflights before `docker build`.

### CI Opt-Out

CI runners without access to the 30 GB noise corpora may set `ACOUSTIC_SKIP_NOISE_PREFLIGHT=1` as a documented, auditable escape hatch. The training host MUST NOT set it. `grep -r ACOUSTIC_SKIP_NOISE_PREFLIGHT` finds every use site in the repo.

### Warning to Future Maintainers

This preflight is intentionally **filesystem-only**. Do NOT add:
- `sf.read()` full-file reads (memory bomb on large FSD50K subset)
- Parquet export / regeneration (that's Plan 20.1-03's contract, and the exporter itself has a known memory issue — see concerns log)
- App imports that could trigger model loading at test collection time

## Carried Concerns

- **[NEW 2026-04-07] `scripts/export_uma16_parquet.py` has an unbounded-memory bug.** `export_ambient()` and `export_eval()` both `path.read_bytes()` every WAV into a Python list before building a single `pa.table()` and writing it out. For the full `data/field/uma16_ambient/` tree this accumulates multi-GB of raw WAV bytes in Python objects simultaneously. An agent running this script during Phase 20.1 validation crashed the dev-host VS Code session. **Resolution:** refactor to stream via `pq.ParquetWriter` with bounded `RecordBatch` flushes (proposed: 64–128 WAVs per batch). Not addressed in this plan (D-19 prohibits touching source-tree scripts from the preflight plan). Should be a Phase 20 hotfix / `/gsd-quick`.
