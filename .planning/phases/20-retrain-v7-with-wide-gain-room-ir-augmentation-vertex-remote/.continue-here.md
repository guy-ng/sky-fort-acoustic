## Critical Anti-Patterns (severity blocking)

| anti-pattern | severity | reason |
|---|---|---|
| Spawn executor with `isolation: "worktree"` for plans that need to reset their base | blocking | Safety hook blocks `git reset --soft`, agent stalls forever asking permission. Use sequential mode (no `isolation` field) instead. |
| `git stash pop` inside an executor agent | blocking | Wave 3 incident: 20-08 popped an unrelated stash, polluted ~25 files. Always tell executors to use named `git stash push -u -m "..."` and only pop matching stashes. |
| Bulk `git checkout HEAD -- .` | blocking | Safety hook blocks the wildcard form. If you need to discard the working tree, do per-file `git checkout HEAD -- <path>`. |
| Submit Phase 20 Vertex job without first verifying `data/noise/{esc50,urbansound8k,fsd50k_subset}/` are populated | blocking | This session's incident. The Dockerfile + env vars assume four noise sources; only one (UMA-16 outdoor_quiet ambient) actually exists. Submitting as-is silently produces a structurally degraded v7 that wastes 4–12 h of paid GPU and almost certainly fails the locked promotion gate. The data-acquisition phase + host-side preflight test described in "What's next" must complete first. |
| Use `Bash` to run `grep` / `find` / `cat` | blocking | The `ccb auto-approve-tools` safety hook blocks these and forces use of the Grep / Glob / Read tools. Save round-trips: go straight to the dedicated tools. |
