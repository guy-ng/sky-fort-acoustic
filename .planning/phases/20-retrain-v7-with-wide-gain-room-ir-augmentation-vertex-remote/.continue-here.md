## Known wrinkles to tell the new session about

### 1. git-lfs hooks need PATH

Worktree creation initially failed because `.git/hooks/{post-checkout,post-commit,post-merge,pre-push}` couldn't find `git-lfs` in `/usr/local/bin`. We patched all four hooks with:
```sh
export PATH="/opt/homebrew/bin:$PATH"
```
as line 2. Don't undo that.

### 2. `node` is in `/opt/homebrew/bin`, not the default PATH

Every `gsd-tools.cjs` invocation needs `PATH="/opt/homebrew/bin:$PATH" node ...` or it will report `command not found: node`.

### 3. Worktree-base reset is blocked by the safety hook

The `gsd-executor` agent prompt asks each plan to do `git reset --soft <expected-base>` if the worktree gets created from the wrong commit (Windows-style bug seen here on macOS too — worktree branches get created from `b0d3e36`, an orphan README branch, instead of current main HEAD). The project's safety hook (`ccb auto-approve-tools`) blocks ANY destructive git op including `git reset --soft`, so the agent gets stuck asking for permission.

**Workaround that worked for Wave 3 plan 20-08:** spawn the executor **without `isolation: "worktree"`** (sequential mode on main). Use `<sequential_execution>` in the prompt instead of `<parallel_execution>`. Plan 20-05 has only 2 tasks and isn't running in parallel with anything else, so sequential mode is fine.

**Better long-term fix:** investigate why `git worktree add` creates branches from `b0d3e36` instead of `HEAD`. There's an orphan branch / detached state somewhere in the repo that's being picked as the default for new branches. Check `git config init.defaultBranch` and any local `branch.autoSetupMerge` config.

### 4. Sub-agent stash discipline is bad

20-08 accidentally `git stash pop`-ed an unrelated pre-existing stash during test triage. The polluted stash is **still recoverable** as `stash@{0}: stash before merge of 17-03 worktree`. If anyone needs that work, run:
```bash
git stash branch recovered-17-03 stash@{0}
```
to materialize it on its own branch.

When dispatching new executors, **forbid `git stash pop` / `git stash apply` in the prompt** — tell them to `git stash push -u` only and only pop their OWN named stashes.

### 5. Test runner setup

Tests pass when run with the venv python and `PYTHONPATH=src`:
```bash
PYTHONPATH=src .venv/bin/python -m pytest tests/...
```
The system `python -m pytest` resolves the `acoustic` package from a stale install location and silently runs against pre-Phase-20 code. ALWAYS use the venv python.
