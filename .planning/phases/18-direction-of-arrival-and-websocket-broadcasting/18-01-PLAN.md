<objective>
Create the DOA coordinate transform module that converts array-frame azimuth/elevation to world-frame pan/tilt degrees, accounting for the UMA-16v2 vertical mounting orientation.

Purpose: Beamforming peaks from multi_peak.py produce (az_deg, el_deg) in the array's internal coordinate system. Downstream PTZ control and WebSocket consumers need pan/tilt in world coordinates. This module bridges that gap with a configurable mounting orientation (D-01, D-02).

Output: `src/acoustic/tracking/doa.py` with `MountingOrientation` enum and `array_to_world()` function; `tests/unit/test_doa.py` with full coverage; `mounting_orientation` setting added to `AcousticSettings`.
</objective>

    from acoustic.tracking.doa import MountingOrientation, array_to_world

    class TestArrayToWorld:
        def test_broadside_identity_vertical(self):

            assert tilt == 10.0

        def test_negative_directions(self):

            assert pan == -45.0
            assert tilt == -20.0

        def test_horizontal_mount_broadside(self):
            pan, tilt = array_to_world(0.0, 0.0, MountingOrientation.HORIZONTAL)
            assert pan == 0.0
            assert tilt == 0.0

        def test_unknown_mounting_raises(self):
            with pytest.raises(ValueError, match="Unknown mounting"):

        def test_enum_values(self):
