## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config -> websocket | ws_targets_hz value controls sleep interval — extreme values could cause tight loop |
| pipeline thread -> async event loop | tracker.update_multi called from pipeline thread, events delivered via call_soon_threadsafe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-06 | Denial of Service | ws_targets_hz set to extreme value | mitigate | Clamp to 0.5-10 Hz range: `max(0.5, min(settings.ws_targets_hz, 10.0))` in websocket loop |
| T-18-07 | Repudiation | Target events without pan/tilt context | accept | Default 0.0 preserves backward compat; no audit requirement for bearing data |
| T-18-08 | Denial of Service | Many concurrent /ws/targets connections at high rate | accept | Existing architecture: each connection is an independent asyncio task; uvicorn connection limits apply |
</threat_model>
