    Create `src/acoustic/evaluation/evaluator.py` with class `Evaluator`:

    - Constructor takes `mel_config: MelConfig | None = None` and `w_max: float = 0.5`, `w_mean: float = 0.5`.
    - Stores a label map: `{"drone": 1, "no drone": 0, "background": 0, "other": 0}` — note "no drone" with space (per D-16). This handles the Real World Testing directory structure.
    - `evaluate(self, model_path: str, data_dir: str) -> EvaluationResult` method:
      1. Load model: `model = ResearchCNN(); model.load_state_dict(torch.load(model_path, weights_only=True)); model.eval()`
      2. Collect files: `paths, labels = collect_wav_files(data_dir, self._label_map)` — reuse existing function
      3. Create aggregator: `aggregator = WeightedAggregator(w_max=self._w_max, w_mean=self._w_mean)`
      4. For each file with `torch.no_grad()`:
         - Read WAV with `soundfile.read(str(path), dtype="float32")`
         - If stereo/multi-channel, average to mono: `audio.mean(axis=1)`
         - Resample if sr != mel_config.sample_rate (use `torchaudio.transforms.Resample`)
         - Extract non-overlapping 0.5s segments (segment_samples = 8000 at 16kHz)
         - For each segment: `mel_spectrogram_from_segment(segment, self._config)` then `model(features).item()`
         - Collect all segment probs, also track p_max and p_mean per file
         - `p_agg = aggregator.aggregate(segment_probs)`
         - Predict: 1 if p_agg >= 0.5 else 0
         - Append FileResult with filename=path.name, true_label, predicted_label, p_agg, correct
      5. Compute metrics via `_compute_metrics(file_results)`:
         - tp/fp/tn/fn from comparing true_label and predicted_label strings
         - accuracy = (tp+tn) / max(tp+tn+fp+fn, 1)
         - precision = tp / max(tp+fp, 1)
         - recall = tp / max(tp+fn, 1)
         - f1 = 2*precision*recall / max(precision+recall, 1e-8)
         - Guard all divisions with max(denominator, 1) to avoid ZeroDivisionError
      6. Compute distribution stats via `_compute_distribution(file_results)`:
         - Separate drone files (true_label == "drone") and background files (true_label == "no_drone")
         - For each class, compute percentiles [25, 50, 75, 95] using `numpy.percentile()` on p_agg, p_max, p_mean arrays
         - If a class has zero files, return DistributionStats(0.0, 0.0, 0.0, 0.0)
      7. Return EvaluationResult with all fields populated

    For per-file p_max and p_mean tracking: store them alongside p_agg in FileResult or as intermediate data. Since D-03 requires per-class distribution of p_max and p_mean, the evaluate loop must track these per file. Add `p_max: float` and `p_mean: float` fields to FileResult (they are needed for distribution stats even though D-02 only shows p_agg in the per-file output).
