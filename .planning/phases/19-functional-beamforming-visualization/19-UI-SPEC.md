## Heatmap Rendering Contract (Phase 19 Specific)

This is the core visual change of Phase 19.

### Before (Current)

- Backend sends `[0, 1]` normalized float32 values via WebSocket
- Frontend applies `v * v` (squared normalization) for visual contrast
- Frontend maps squared value to 256-entry jet colormap LUT

### After (Phase 19)

- Backend applies functional beamforming (`(map / max) ** nu`) with nu=100 default
- Backend sends `[0, 1]` normalized float32 values via WebSocket (same wire format)
- Frontend maps value **directly** to 256-entry jet colormap LUT -- **no squaring**
- Colormap LUT is unchanged (jet, 256 entries, pre-built in `useRef`)

### Visual Contract

| Property | Value |
|----------|-------|
| Colormap | Jet (blue-cyan-green-yellow-red), 256 entries |
| Value range | `[0.0, 1.0]` float32 from backend |
| Frontend transform | None -- direct `Math.round(v * 255) * 3` LUT index |
| Background (no signal) | Values near 0.0 map to deep blue (`#000080`) |
| Peak (source detected) | Value 1.0 maps to dark red (`#800000`) |
| Canvas rendering | `imageRendering: pixelated` (nearest-neighbor upscaling) |
| Canvas sizing | Square, fit to container via ResizeObserver |
| Frame rate | 20 Hz (unchanged, set by backend WebSocket push rate) |

### Wire Format (Unchanged)

| Property | Value |
|----------|-------|
| Protocol | WebSocket `/ws/heatmap` |
| Handshake | JSON `{ width, height, az_min, az_max, el_min, el_max }` |
| Data frames | Binary `Float32Array`, length = width * height |
| Byte order | Little-endian (native) |
| Normalization | `[0.0, 1.0]` -- backend owns all contrast/normalization |

## Interaction Contract

### No New Interactions

Phase 19 introduces zero new user interactions on the frontend. The visual output of the heatmap changes because the backend now produces cleaner, sidelobe-suppressed maps. The user experience improvement is passive: sharper peaks, cleaner background.

### Future Consideration (Not In Scope)

A UI control for adjusting `bf_nu` at runtime could be added in a future phase. The backend PATCH `/api/settings` endpoint created in this phase enables that future work. This phase does NOT include a frontend nu slider or control.
